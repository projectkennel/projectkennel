# docs/reference — machine-coupled and as-built artefacts

The corpus (design + Linux realisation) lives in the **two-volume book**,
`github.com/projectkennel/books`, checked out in-tree at `books/` (gitignored). This directory is
what stays *in this repository* because it is machine-coupled to the code or records what the built
system actually does — neither of which belongs in the design narrative:

- **`THREATS.md`** — the canonical threat catalogue (prose). Machine-coupled: `dist/threats/catalogue.toml`
  is its machine form (what `kennel policy risks` reads), kept in sync by a CI guard
  (`src/tools/tests/threats-catalogue.sh`); compile diagnostics and the issue-tag Action cite its
  T-IDs. The book's Threats material derives from this file; it never forks it.
- **`crate-inventory.json`** and **`03-crate-decomposition.md`** — the crate inventory and SLOC table,
  regenerated by `gen-inventory` and diff-checked in CI. Machine-generated; the generator is the
  source of truth.
- **`AS-BUILT.md`** — the durable record of as-built facts (what the running code does), the
  steady-state successor of the freeze-era patch log. Kept deliberately separate from the book: the
  book states the design, this log states where the build elaborates or diverges from it.

The authoritative policy schema is generated to `schema/policy.toml.schema` (derived from the parser
structs); it is not duplicated here.
